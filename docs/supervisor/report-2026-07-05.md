*Surfaced by the Alignment Checker (during aggregation of the first dogfood run).*

The supervisor implementation consistently references `./docs/knowledge_index.md` as the report index — but the project's knowledge index actually lives at **`./docs/knowledge/knowledge_index.md`**. The mismatch spans three APPROVED Tasks: `SKILL.md` § /shannon-report contract + § Report Pipeline step 6 (TASK-011/015), the `footer.md` template (TASK-015), and — most consequentially — the PreToolUse write-guard's exception (`pretool-writeguard.sh` line 18: `index_exception="docs/knowledge_index.md"`, TASK-016). This is a genuine integration defect that each isolated Task's Gate-3 verification could not catch (they checked internal consistency, not the real filesystem path) — the first end-to-end run is what surfaced it.
